# Skill: write-and-update-prd

## When to Use
Use this skill when:
- The user requests a major new feature, service description, or route.
- Changing core product logic, integration specs, or database columns.
- Performing architectural pivots.

## Principles
1. **Sync Before Code**: The PRD must be updated and approved *before* writing source code changes.
2. **Business Case Alignment**: Every requirement must align with the Smesh.dev agency positioning (automations, n8n, AI, and static performance).
3. **Traceability**: Link requirements to implementation tickets.

## Task Checklist
- [ ] Update the target section in `production_artifacts/PRD.md`.
- [ ] Highlight the new feature goals, user stories, and technical requirements.
- [ ] Request feedback and approval from the user before executing code changes.
